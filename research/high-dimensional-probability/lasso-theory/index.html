Lasso Theory

{% include research-topic-nav.html back_label="Back" back_url="/research/high-dimensional-probability/" prev_label="Last page" prev_url="/research/high-dimensional-probability/blessing-of-dimensionality/" %}

The lasso estimates a sparse regression vector when the number of covariates \(p\) may be much larger than the sample size \(n\).

Model and estimator

In the linear model \(y=X\beta^0+\varepsilon\), the lasso estimator is

\[\hat\beta=\arg\min_{\beta\in\mathbb R^p}\left\{\frac{1}{2n}\|y-X\beta\|_2^2+\lambda\|\beta\|_1\right\}.\]

The \(\ell_1\) penalty encourages sparse solutions and is convex, so the optimization problem is computationally tractable.

Basic inequality

By optimality of \(\hat\beta\), with \(\Delta=\hat\beta-\beta^0\),

\[\frac{1}{2n}\|X\Delta\|_2^2+\lambda\|\hat\beta\|_1\le \frac{1}{n}\varepsilon^T X\Delta+\lambda\|\beta^0\|_1.\]

If \(\lambda\ge 2\|X^T\varepsilon/n\|_\infty\), this inequality forces \(\Delta\) to lie in a cone around the true support \(S=\{j:\beta_j^0\ne0\}\).

Oracle rate

Under a restricted eigenvalue condition and \(|S|=s\), one obtains

\[\frac{1}{n}\|X(\hat\beta-\beta^0)\|_2^2 \lesssim s\lambda^2,\qquad \|\hat\beta-\beta^0\|_1\lesssim s\lambda.\]

For sub-Gaussian noise, \(\lambda\asymp \sqrt{\log p/n}\), so the prediction error is of order \(s\log p/n\).

Core message

The lasso works when the true signal is sparse and the design is not too collinear on sparse directions. The price for searching among \(p\) variables is logarithmic in \(p\).

Théorie du lasso

{% include research-topic-nav.html back_label="Retour" back_url="/research/high-dimensional-probability/" prev_label="Page précédente" prev_url="/research/high-dimensional-probability/blessing-of-dimensionality/" %}

Le lasso estime un vecteur de régression parcimonieux lorsque le nombre de variables \(p\) peut dépasser largement la taille d'échantillon \(n\).

Modèle et estimateur

Dans le modèle linéaire \(y=X\beta^0+\varepsilon\), l'estimateur lasso est

\[\hat\beta=\arg\min_{\beta\in\mathbb R^p}\left\{\frac{1}{2n}\|y-X\beta\|_2^2+\lambda\|\beta\|_1\right\}.\]

La pénalisation \(\ell_1\) favorise la parcimonie et garde le problème convexe.

Inégalité de base

Par optimalité de \(\hat\beta\), avec \(\Delta=\hat\beta-\beta^0\),

\[\frac{1}{2n}\|X\Delta\|_2^2+\lambda\|\hat\beta\|_1\le \frac{1}{n}\varepsilon^T X\Delta+\lambda\|\beta^0\|_1.\]

Si \(\lambda\ge 2\|X^T\varepsilon/n\|_\infty\), cette inégalité impose que \(\Delta\) reste dans un cône autour du vrai support \(S=\{j:\beta_j^0\ne0\}\).

Taux oracle

Sous une condition de valeur propre restreinte et avec \(|S|=s\), on obtient

\[\frac{1}{n}\|X(\hat\beta-\beta^0)\|_2^2 \lesssim s\lambda^2,\qquad \|\hat\beta-\beta^0\|_1\lesssim s\lambda.\]

Pour un bruit sous-gaussien, \(\lambda\asymp \sqrt{\log p/n}\), donc l'erreur de prédiction est d'ordre \(s\log p/n\).

Message central

Le lasso fonctionne lorsque le signal est parcimonieux et que le plan n'est pas trop colinéaire sur les directions parcimonieuses. Le coût de chercher parmi \(p\) variables est logarithmique en \(p\).

Lasso 理论

{% include research-topic-nav.html back_label="返回" back_url="/research/high-dimensional-probability/" prev_label="上一页" prev_url="/research/high-dimensional-probability/blessing-of-dimensionality/" %}

Lasso 用于在 \(p\) 可能远大于 \(n\) 的线性回归中估计稀疏系数向量。

模型与估计量

在线性模型 \(y=X\beta^0+\varepsilon\) 中，lasso 估计量定义为

\[\hat\beta=\arg\min_{\beta\in\mathbb R^p}\left\{\frac{1}{2n}\|y-X\beta\|_2^2+\lambda\|\beta\|_1\right\}.\]

\(\ell_1\) 惩罚鼓励稀疏解，同时保持优化问题为凸问题。

基本不等式

由 \(\hat\beta\) 的最优性，令 \(\Delta=\hat\beta-\beta^0\)，有

\[\frac{1}{2n}\|X\Delta\|_2^2+\lambda\|\hat\beta\|_1\le \frac{1}{n}\varepsilon^T X\Delta+\lambda\|\beta^0\|_1.\]

若 \(\lambda\ge 2\|X^T\varepsilon/n\|_\infty\)，则该不等式迫使 \(\Delta\) 落在真实支撑集 \(S=\{j:\beta_j^0\ne0\}\) 附近的锥内。

Oracle 速度

若 restricted eigenvalue 条件成立且 \(|S|=s\)，则

\[\frac{1}{n}\|X(\hat\beta-\beta^0)\|_2^2 \lesssim s\lambda^2,\qquad \|\hat\beta-\beta^0\|_1\lesssim s\lambda.\]

若噪声为 sub-Gaussian，取 \(\lambda\asymp \sqrt{\log p/n}\)，预测误差量级为 \(s\log p/n\)。

核心信息

Lasso 有效的关键是：真实信号稀疏，且设计矩阵在稀疏方向上不太共线。在 \(p\) 个变量中搜索的代价通常只以 \(\log p\) 的形式出现。
