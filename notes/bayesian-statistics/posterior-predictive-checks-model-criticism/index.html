9. Posterior predictive checks and model criticism

{% include note-topic-nav.html back_label="Back" back_url="/notes/bayesian-statistics/" prev_label="Last page" prev_url="/notes/bayesian-statistics/robust-models-missing-data-data-collection/" next_label="Next page" next_url="/notes/bayesian-statistics/model-comparison-cross-validation-bayes-factors/" %}

Posterior predictive checks ask whether data replicated from the fitted model resemble the observed data in scientifically relevant ways. They are model criticism, not proof that the model is true.

Replicated data

A posterior predictive check compares the observed data with data generated after conditioning on the observed data.

Definition. Let \(Y^{\mathrm{rep}}\) be drawn from \[ p(y^{\mathrm{rep}}\mid y)=\int p_\theta(y^{\mathrm{rep}})\Pi(\mathrm{d}\theta\mid y). \] For a discrepancy \(T\), the posterior predictive tail probability is \[ p_{\mathrm{ppc}}=\mathbb P\{T(Y^{\mathrm{rep}},\theta)\ge T(Y,\theta)\mid Y\}. \]

Remark. The discrepancy may depend on \(\theta\), but then both replicated and observed discrepancies must be computed under the same posterior draw of \(\theta\).

Double use of data

Posterior predictive \(p\)-values are usually conservative because the data are used both to fit and to check.

Proposition. Posterior predictive \(p\)-values are not generally uniform under the model.

Proof. Uniformity would require the reference distribution of the statistic to be fixed before seeing the data. Here the reference distribution is itself conditioned on the observed data, which pulls replications toward the observed features and reduces tail variation.

Graphical checks and residual structure

The most useful checks are often graphical: residual patterns, tail behavior, group discrepancies, and dependence not captured by the likelihood.

Example. In a regression model, one may compare residual maxima, residual autocorrelation, or groupwise variance between \(Y\) and \(Y^{\mathrm{rep}}\). A systematic mismatch points to a missing component of the likelihood, not merely to a bad parameter estimate.

Checks are conditional on the question

A model can pass one check and fail another. That is a feature, because different checks target different scientific failures.

Remark. Posterior predictive checking should be read alongside prior predictive checking: the first asks whether the fitted model can reproduce the observed data; the second asks whether the model was plausible before seeing the data.
