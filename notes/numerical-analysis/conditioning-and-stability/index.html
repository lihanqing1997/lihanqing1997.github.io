Conditioning belongs to the mathematical problem; stability belongs to the algorithm. Numerical analysis is careful about keeping these two ideas separate.

Absolute and relative condition numbers

Definition. For a differentiable map \(\Phi:X\to Y\), the absolute condition number at \(x\) is \(\|D\Phi(x)\|\). The relative condition number is \(\kappa_{\mathrm{rel}}(x)=\|D\Phi(x)\|\|x\|/\|\Phi(x)\|\), when the denominator is nonzero.

Stability

Definition. An algorithm is forward stable if its forward error is comparable to the unavoidable conditioning error. It is backward stable if its computed output is the exact output for a small perturbation of the input.

Conditioning-stability principle

Theorem. If an algorithm is backward stable and the problem has relative condition number \(\kappa\), then the relative forward error is typically bounded by a constant times \(\kappa u\), up to higher-order terms. Thus poor conditioning cannot be repaired by a stable algorithm; instability can ruin even a well-conditioned problem.

Residual versus error

Proposition. A small residual implies a small forward error only after multiplication by an appropriate condition number. For linear systems, \(\|r\|/\|b\|\) controls \(\|\widehat x-x\|/\|x\|\) through \(\kappa(A)\), not by itself.

Diagnostic interpretation

Remark. When a computation fails, the first question is whether the problem is ill conditioned or the algorithm is unstable. These require different fixes: reformulate or regularize the problem in the first case, change the algorithm in the second.
