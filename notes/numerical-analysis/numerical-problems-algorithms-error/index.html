1. Numerical problems, algorithms, and error

{% include note-topic-nav.html back_label="Back" back_url="/notes/numerical-analysis/" next_label="Next page" next_url="/notes/numerical-analysis/floating-point-arithmetic/" %}

Numerical analysis begins by separating the mathematical problem, the algorithm used to approximate it, and the several notions of error by which the approximation is judged.

Numerical problem

Definition. A numerical problem is a map \(\Phi:X\to Y\) between normed spaces, together with a class of admissible data \(x\in X\). An algorithm replaces \(\Phi(x)\) by a computable object \(A(x)\), often after discretizing \(X\), \(Y\), or both.

Forward error and residual

Definition. The forward error is \(\|A(x)-\Phi(x)\|_Y\). If the target is characterized by an equation \(F(y,x)=0\), the residual is \(\|F(A(x),x)\|\). A small residual is meaningful only when the equation is well conditioned.

Backward error

Definition. The backward error of an approximation \(\widehat y\) is the smallest perturbation of the data for which \(\widehat y\) is exact: \(\inf\{\|\Delta x\|: \Phi(x+\Delta x)=\widehat y\}\). Backward error is often the cleanest way to express algorithmic stability.

Error decomposition

Proposition. In a finite computation the total error is usually decomposed as approximation error plus propagated rounding error. For a discretization parameter \(h\) and floating-point unit roundoff \(u\), one often obtains a bound of the form \(C_1h^p+C_2u h^{-q}\), which explains why simply making \(h\) smaller can eventually make the answer worse.

Proof by comparison

Proof. A typical numerical proof compares three objects: the exact continuum solution, the exact discrete solution, and the computed discrete solution. The first gap is controlled by approximation theory; the second by stability of the discrete algorithm.
