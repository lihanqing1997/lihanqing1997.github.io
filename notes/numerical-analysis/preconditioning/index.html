18. Preconditioning

{% include note-topic-nav.html back_label="Back" back_url="/notes/numerical-analysis/" prev_label="Last page" prev_url="/notes/numerical-analysis/krylov-subspace-methods/" next_label="Next page" next_url="/notes/numerical-analysis/fixed-point-iteration-bisection/" %}

Preconditioning changes the geometry of a linear system so that an iterative method sees a better conditioned problem.

Preconditioned system

Definition. A preconditioner \(M\) approximates \(A\) but is easier to invert. One may solve \(M^{-1}Ax=M^{-1}b\), \(AM^{-1}y=b\), or a symmetrically preconditioned equivalent system.

Spectral clustering

Proposition. For Krylov methods, a good preconditioner clusters eigenvalues or reduces the effective condition number of the preconditioned operator. The preconditioner should also be cheap enough that each iteration remains affordable.

Incomplete factorization

Definition. Incomplete LU or Cholesky factorization drops selected fill-in entries while approximating a direct factorization. The dropping rule balances memory, stability, and quality of approximation.

Multigrid principle

Theorem. For many elliptic discretizations, smoothers reduce high-frequency error while coarse-grid correction reduces low-frequency error. Combining these effects can yield convergence rates essentially independent of mesh size.

Problem dependence

Remark. There is no universal preconditioner. Effective preconditioning uses structure: ellipticity, graph geometry, block variables, low-rank components, or statistical covariance models.
