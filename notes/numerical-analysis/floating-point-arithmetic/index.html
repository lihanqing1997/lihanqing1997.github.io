Floating-point arithmetic is the model that explains why algebraically equivalent formulas may have very different numerical behavior.

Floating-point system

Definition. A floating-point system stores numbers as \(\pm m\beta^e\), where the base \(\beta\), precision, and exponent range are finite. The rounding map \(\mathrm{fl}\) sends a real number to a nearby representable number.

Standard rounding model

Lemma. For a normalized operation \(\circ\in\{+,-,\times,/\}\), the computed result is modeled as \(\mathrm{fl}(a\circ b)=(a\circ b)(1+\delta)\), with \(|\delta|\le u\), provided no overflow or underflow occurs. The constant \(u\) is the unit roundoff.

Catastrophic cancellation

Proposition. Subtracting nearly equal numbers can turn small relative input errors into large relative output errors. If \(x\approx y\), then the relative condition of \(x-y\) with respect to perturbations in \(x,y\) is proportional to \((|x|+|y|)/|x-y|\).

Stable summation

Remark. Summing in a fixed order may lose small terms when magnitudes vary. Pairwise summation and compensated summation reduce error growth; the point is not that rounding disappears, but that the rounding errors are prevented from accumulating in the worst possible order.

Backward stability of elementary operations

Proof. The standard rounding model can be rewritten as an exact operation on slightly perturbed inputs. This observation is the seed of backward error analysis: a computed answer is accepted when it solves a nearby problem and the nearby problem is statistically or physically indistinguishable from the original one.
