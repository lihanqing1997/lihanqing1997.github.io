12. Prefix codes and Kraft-McMillan inequality

{% include note-topic-nav.html back_label="Back" back_url="/notes/information-theory/" prev_label="Last page" prev_url="/notes/information-theory/information-spectrum/" next_label="Next page" next_url="/notes/information-theory/lossless-source-coding-theorem/" %}

Lossless coding begins with a purely combinatorial constraint: codeword lengths cannot be arbitrary if unique decoding is required.

Prefix code

Definition. A binary code \(c:\mathcal X\to\{0,1\}^*\) is prefix-free if no codeword is a prefix of another. Its length function is \(\ell(x)=|c(x)|\).

Kraft-McMillan inequality

Theorem. A set of integer lengths \(\{\ell(x):x\in\mathcal X\}\) is realized by a binary prefix code only if \[ \sum_{x\in\mathcal X}2^{-\ell(x)}\le1. \] Conversely, every integer length sequence satisfying this inequality can be realized by a prefix code.

Proof by binary trees

Proof. A prefix code corresponds to leaves in the infinite binary tree. A leaf at depth \(\ell\) occupies fraction \(2^{-\ell}\) of the boundary. Prefix-freeness makes these boundary cylinders disjoint, giving the inequality. The converse constructs leaves greedily in lexicographic order whenever the remaining Kraft mass permits it.

Expected length lower bound

Corollary. For any prefix code for \(X\), \[ \mathbb E\ell(X)\ge H_2(X), \] where \(H_2\) is entropy in bits. The proof compares \(P(x)\) to the distribution proportional to \(2^{-\ell(x)}\) and uses nonnegativity of relative entropy.
