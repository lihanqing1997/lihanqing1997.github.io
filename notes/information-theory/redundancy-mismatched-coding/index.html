14. Redundancy and mismatched coding

{% include note-topic-nav.html back_label="Back" back_url="/notes/information-theory/" prev_label="Last page" prev_url="/notes/information-theory/lossless-source-coding-theorem/" next_label="Next page" next_url="/notes/information-theory/universal-source-coding-minimax-redundancy/" %}

Relative entropy becomes concrete in coding: it is the excess expected length paid when a code is designed for the wrong distribution.

Ideal lengths

Definition. For a design distribution \(Q\), the ideal real-valued code length in nats is \(\ell_Q(x)=-\log Q(x)\). Integer prefix lengths add only rounding terms to this ideal picture.

Redundancy identity

Proposition. If the true law is \(P\) but lengths are matched to \(Q\), then \[ \mathbb E_P\ell_Q(X)=H(P)+D(P\|Q). \] Thus relative entropy is the exact ideal excess length.

Proof by expansion

Proof. Expand \[ \mathbb E_P[-\log Q(X)]=\sum_xP(x)\left[-\log P(x)+\log\frac{P(x)}{Q(x)}\right] =H(P)+D(P\|Q). \]

Model selection interpretation

Remark. Coding redundancy is the information-theoretic face of likelihood regret. Choosing a model class asks for short codes uniformly over possible sources, which leads naturally to universal coding and MDL principles.
