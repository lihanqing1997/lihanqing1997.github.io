Measures, extension, and monotone classes

{% include probability-note-nav.html back_label="Back" prev_label="Last page" prev_url="/notes/probability-theory/probability-spaces/" next_label="Next page" next_url="/notes/probability-theory/distributions-random-variables/" %}

This topic belongs to the measure-theoretic foundation of probability. The main purpose is to make the objects measurable before one starts computing. Here the focus is Measures, extension, and monotone classes.

Start with the precise object. The definition states what the object is and what assumptions make the later formulas meaningful.

Definition. A \(\pi\)-system is a class closed under finite intersections. A \(\lambda\)-system contains \(\Omega\), is closed under relative complements of nested sets, and is closed under countable disjoint unions. These classes let us extend identities from generators to generated \(\sigma\)-fields.

These assumptions are part of the mathematics. They tell us where each expression lives and prevent informal notation from hiding a real domain issue.

The result below is the fact one usually uses in calculations or proofs. Read it as the operational content of the definition.

Result. If \(\mathcal C\) is a \(\pi\)-system, then the smallest \(\lambda\)-system containing \(\mathcal C\) is \(\sigma(\mathcal C)\). Hence two probability measures that agree on \(\mathcal C\) agree on \(\sigma(\mathcal C)\).

Proof.

The proof is easiest to read by separating the elementary case from the extension step.

Let \(\mathcal L\) be the class of sets on which two measures agree. If they agree on \(\mathcal C\), then \(\mathcal C\subset\mathcal L\). Finite additivity and continuity from below show that \(\mathcal L\) is a \(\lambda\)-system. Since \(\mathcal C\) is a \(\pi\)-system, \(\sigma(\mathcal C)\subset\mathcal L\).

Remarks

This is the standard way to prove identities: check rectangles, intervals, or cylinder sets first, then extend by a monotone class argument.
